# 精訊版《勇者鬥惡龍 III》音樂保存紀錄(台灣電玩史)

> 一份保存紀錄:把 1993 年精訊資訊未發售的中文化《勇者鬥惡龍 III》裡的遊戲音樂,
> 從原版執行檔反組譯、抽出、並用合成器重新發聲,留給後人。
> 做法、技術細節見 [`57-cmf-audio-re.md`](57-cmf-audio-re.md);本檔記錄「為什麼、是什麼、保存了什麼」。

## 背景

1990 年代初,台灣的**精訊資訊(King Information)**著手把 ENIX 的《勇者鬥惡龍 III》(1988, FC)
移植並中文化到 MS-DOS PC 上,中文標題作《傳說的終章》。這個版本**最終未正式發售**,
留下一份接近完成、但帶有缺口(部分怪物缺圖、音樂與部分流程未完全接好)的 build。

那個年代的台灣電玩產業,常以這種「把日本經典移植 + 全面中文化」的方式累積技術:
自製點陣中文字庫、注音輸入、DOS 下的圖文引擎、聲霸卡音訊驅動整合。精訊這份 DQ3 正是其中一個樣本——
它沒有上市,卻保存了那個時期台灣工程師的工藝。本專案的目的,就是把這份工藝**完整還原並保存**。

## 音樂系統(精訊當年怎麼做的)

反組譯確認,精訊版的聲音架構是當年 DOS 遊戲的主流組合:

| 項目 | 內容 |
|---|---|
| 音源 | **Creative Sound Blaster 的 OPL2 FM 合成**(YM3812 晶片;AdLib/聲霸卡音色) |
| 音樂驅動 | Creative `SBFM_CMF_DRV`(CMF 格式 FM 音樂驅動,連結自 `SBCM.LIB`) |
| 音樂資料 | 外部封包檔 **`MBG.MCX`**(127 KB):前導偏移表 + **18 條** CMF 變體音軌 |
| 音效 | Creative VOC 數位音(`FVOC.VCX` 22 個 + `NVOC.VCX` 24 個,共 **46 個**音效) |
| 不支援 | **沒有 MT-32 / Roland**(無 MPU-401 埠操作);當年只做了聲霸卡 FM 一種音源 |

音樂事件流本身是 MIDI 風格的(note on/off、program change、多聲道 + running status),
由驅動即時餵給 OPL2 FM 晶片發聲。這是那個時代「用 2 顆 operator 的 FM 合成出整首曲子」的典型手法。

## 保存了什麼

1. **18 條原生 OPL2 FM 音樂**——從 `MBG.MCX` 抽出、用重建的 OPL2 FM 合成器(`dq3_opl2`)重新發聲,
   還原當年聲霸卡的音色與精訊自己編寫的旋律編排。
2. **46 個數位音效**——`FVOC/NVOC.VCX` 的 Creative VOC PCM,解碼後可播放。
3. **兩種版本的離線錄音**(供保存與聆聽):
   - **SB 版**:精訊原生的聲霸卡 OPL2 FM 音色(忠於當年實機聽感)。
   - **MT-32 / Roland 風版**:把事件流轉成標準 MIDI、用 General MIDI 音色庫合成的管弦/Roland 風重新詮釋
     ——這是精訊**當年沒有做**的音源,屬於「如果當年有 Roland 會是什麼樣子」的再想像,非歷史原貌,標示清楚以免混淆。

## 方法(如何從一個 30 年前的執行檔裡把音樂挖出來)

全程靜態反組譯,不靠猜測(細節見 docs/57):

1. 解析 Creative 驅動庫 `SBCM.LIB`(OMF 物件庫格式),取出 FM/CMF 驅動的機器碼。
2. 以 byte-match 把驅動定位進 `DQ3.EXE`(16-bit 大模式 DOS 程式),解出段位址換算公式。
3. 反追到遊戲呼叫 `_sbfm_play_music` 的位置,解出它從 `MBG.MCX` seek 載入音樂的機制。
4. 解碼音軌格式(MIDI-like 事件 + running status + tempo),重建 OPL2 FM 合成器播放。

## 為什麼留這份紀錄

精訊資訊這款 DQ3 沒有上市,在公開資料裡幾乎只剩零星的遊戲錄影。那個年代台灣團隊在 DOS 上自製
中文字庫、注音、FM 音訊整合的工藝,若不主動保存,會隨著磁片與機器一起消失。

把這 18 首當年沒能讓玩家在零售盒裝裡聽到的曲子重新發聲、留成可保存的檔案,
是對那段台灣電玩史、以及對當年那群工程師的一個紀念。
