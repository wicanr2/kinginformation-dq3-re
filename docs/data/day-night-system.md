# 晝夜系統 RE 確認(第一性原理,2026-06-27)

> 使用者問:「遊戲是否有黑夜系統(野外晃會白天/黃昏/黑夜/黎明轉換)?」用第一性原理 + RE 確認。

## 結論

| 問題 | 答案 |
|---|---|
| 原版有沒有晝夜**狀態**? | **有**(白天/夜晚兩態,影響事件)|
| 走動會**自動**循環時間(白天→黃昏→黑夜→黎明)? | **不會**(非步數驅動)|
| 晝夜由什麼切換? | **ラナルータ 拉那魯達(咒文 rec177,按需切換)+ 黑暗之燈(道具 0x5f,變夜)** |
| remake 現況 | **未實作**(無晝夜狀態、無切換、無夜晚 palette)|

精訊版是「白天/夜晚**兩態切換**」,**不是** DQ5/6 那種隨步數連續流逝的時刻系統。野外閒晃不會變天。

## 第一性原理論證

### 1. 設計反證(柵欄原則):黑暗之燈的存在本身就是證明
- `黑暗之燈`(item 0x5f)的**唯一用途 = 強制變夜**(docs/re-log-722:215/436、quest-items.md:Ch22「變夜」)。
- **若走動會自動入夜,這道具毫無存在意義** —— 玩家只要原地走幾步就入夜了。
- 一個「專門用來把場景變成夜晚」的道具存在,正反證**夜晚不會自動到來**,必須靠道具/咒文促成。

### 2. ラナルータ = 「按需切換」咒文
- `拉那魯達 ラナルータ`(rec177,BBS 史料 §九「晝夜轉換」)是玩家主動施放的**切換**咒。
- 怪物(笑袋 39 / 蝙蝠貓 54 / 殭屍魔人 110,docs/38)戰鬥中也會施 ラナルータ → 戰中切晝夜。
- 「需要一個咒文來切換」本身說明晝夜是**離散、被觸發**的狀態,非自動流逝。

### 3. RE:每步移動 handler 不碰時間計數器
- 反組譯每步 gameplay loop(docs/11 §4,`bp=[0x4f46]` 事件旗標逐位元 dispatch):
  處理「觸發 tile / 倒數計時器 `[0x52f6/7/8]`(聖水類暫時效果)/ tile 動畫 `[0x2577]` / NPC 迴圈」。
- **沒有任何「時間 / 晝夜 / 時刻」計數器遞增**。`[0x52f6..8]` 是聖水驅敵之類的暫時 step 倒數,歸 0 即效果結束,
  與晝夜無關。⇒ 走一步不會推進任何時刻時鐘。
- 遭遇也只是每步擲骰(docs/39),同樣不寫時間。

### 4. 劇情依「晝夜兩態」設計(佐證狀態存在但離散)
- 劇本(`docs/script/txt01/03/04/05`)多處:「只能在白天」「等待夜晚的過去」「每當夜晚來臨」
  「在夜晚會發生…」「就這樣,夜晚過去了」。
- 提頓村牢房:**夜晚進村 + 開牢門**才觸發(綠寶珠 questline,re-log-722:215)。
- 這些是「白天/夜晚」二選一的條件分支,不是「下午三點」式連續時刻 → 印證離散兩態。

## remake 待實作(若要補)

remake 目前無晝夜。要忠實補上,最小集:
1. 晝夜狀態 global(`g_is_night` 0/1)+ 存檔欄。
2. **ラナルータ(rec177)** 野外咒文 → toggle(接進 `field_cast_modal`)。
3. **黑暗之燈(0x5f)** 道具使用 → set night(接進 item-use)。
4. 夜晚 overworld/城鎮用較暗 palette(程式調暗 DQ3.PAL,類似海面 cycling 機制)。
5. 夜晚 gated 事件(提頓村牢房等)讀 `g_is_night`。
