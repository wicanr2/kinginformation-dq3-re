# 精訊版《勇者鬥惡龍 III》1994–1995 BBS 討論串(歷史紀錄)

這份文件整理自一份當年的 BBS 討論串存檔(`dq3_bbs/DQ3.pdf`,30 頁),內容是 1994 年到
1995 年間,台灣各 FidoNet / 90Net / Gamenet 站台上玩家對「精訊版 DQ3」的攻略、bug 修改、
存檔修改與物品咒文整理。原始 PDF 不入版控(著作權考量),這裡保留**經整理的文字內容**作為
專案的歷史脈絡與一手史料。

精訊版 DQ3 從未正式發售,當年只在 BBS 上以未完成版本流傳。這串討論是它存在過、被一整代玩家
玩過、抱怨過、破解過的證據。對本反組譯專案而言,它同時是**一手的 bug 對照表**——當年玩家用
hex editor 摸出來的修改法,正好對得上我們今天從 `DQ3.EXE` 反組譯出的同一批 bug。

---

## 一、精訊資訊的官方公告(1994.05.23)

討論串裡有人轉貼了精訊資訊張貼在自家 BBS 上的公告。這是這款遊戲身世的第一手說明:

> **敬告各界及 BBS 站:**
>
> 本公司於 1993 年設計研發的中文電腦遊戲「勇者鬥惡龍Ⅲ」,因日本 ENIX 公司不同意在台灣
> 發售,本公司已中止此一遊戲的開發。近日有本公司不肖離職員工將此一未開發完整的版本流傳至
> 坊間的 BBS 站,令一般的電腦使用者誤以為此為 SHAREWARE 軟體,嚴重侵犯日本 ENIX 公司
> 及本公司著作權,本公司將追究此一事件之責任外,並呼籲各 BBS 站及使用者切勿再任意拷貝、
> 陳列以及散佈本軟體,以避免觸犯著作權。
>
> &nbsp;&nbsp;&nbsp;&nbsp;精訊資訊有限公司　董事長　李培民

幾件事這份公告替我們定錨了:

- **開發年份 1993**,中止於授權問題(ENIX 不同意在台發售)。
- 流出版本是**未開發完整的版本**——這解釋了我們在反組譯中看到的種種未完成痕跡(九頭龍 /
  歐里狄加 sprite 空缺、寶箱開不開、彩虹水滴合成產出錯物品)。
- 公司是**精訊資訊**,董事長**李培民**。CTY0 sec0 左上角作者留下的 `LIH` 彩蛋(見 README)
  即出自這個開發團隊的時代。

另一則貼文補充了流傳經過:「精訊在去年十月本來要發售的『勇者鬥惡龍 3』,聽說因為圖形名稱
授權問題而流標了,並且放在 InterNET 上到處傳。」(Werong Ho, 1994.05.11)

## 二、職業特性(Dollar Cheng 故事攻略開場)

最完整的一篇攻略出自 **Dollar Cheng【孔方兄】**(1994.07.15,90-GAME),開頭先講八職業特性。
對照我們從成長表(docs/23)反推的職業數值傾向,當年玩家的體感描述完全吻合:

| 職業 | 當年描述重點 | 對照 RE 成長表(docs/23) |
|---|---|---|
| 勇者 | 世界唯一、不能轉職 / 存登記處;裝備與魔法最平均,有專用裝備魔法 | 各屬性中庸;聰明(智)成長低 |
| 戰士 | 攻防高、裝備貴、速度低、不能用魔法 | 力量 slope 高(20)、聰明=0、速度低 |
| 武鬥家 | 不需裝備、空手最強、速度快、高機率連續會心一擊 | 力量最高(slope 22)、速度最高(slope 9) |
| 僧侶 | 神父系;回復 & 復活咒文為主,兼攻擊輔助 | 聰明(智)成長高、體力中上 |
| 魔法使 | 攻擊咒文專家;裝備少、體力弱 | 聰明最高、體力(耐力)成長靠咒文撐 |
| 賢者 | 兼僧侶系 & 魔法使系咒文、裝備多;升級慢需下工夫 | 全能型、門檻表偏高 |
| 商人 | 鑑定 / 賣價能力、戰後多金;初期像勇者 | 平均型 |
| 遊閒者(遊玩人) | 運氣好、增添樂趣;等級足夠可輕易轉賢者 | 運氣(luck)slope 最高(10) |

作者建議新手隊伍:「勇戰僧魔」或「勇武僧魔」。

## 三、故事攻略(Dollar Cheng,1994.07)

> 摘要保留主線脈絡與精訊版專屬差異;地名 / 物品名沿用當年譯名。

遊戲由勇者家中床上醒來開始,母親帶你進城晉見國王,國王發部分裝備並指示去**城外酒店挑選同伴**
(即露依達酒場;對照 remake 的露依達酒場創角)。阿里阿罕大陸有一城、一村、一塔、四洞窟、一祠堂。

主線推進:雷貝村南 / 阿里阿罕西南洞窟 → 小島**拿米基之塔**塔頂取**盜賊的鑰匙** → 雷貝村東北
(有馬處)開門上樓取**魔法玉** → 東南「誘人的洞窟」用魔法玉(炸彈)炸牆 → 經**旅人之門**(旅人
之泉 / 時空之門)到**羅馬利亞城**(找回遭竊皇冠)。北方一帶有祠堂、卡薩布村、香巴尼塔、諾亞尼爾
村、愛路夫隱處與洞窟。

關鍵精訊版差異(攻略作者反覆強調):

- **遇敵機率過高**:「碰上敵人的機率頗大」;黃金爪會再增加遭遇率,建議賣掉或寄放。
- **寶箱 bug**:有魔法鑰匙後可開阿里阿罕城 & 依西絲城寶箱,「但是精訊版的 DQ3 有 BUG,有些寶箱
  可能會打不開」。
- **沙曼歐沙城南洞窟**:寶箱被精訊改成「所有寶箱都是米米克」(除拉之鏡 & 貓衣外)。
- **彩虹水滴合成 bug**:太陽石 + 雲雨之杖在利姆達爾南祠堂「居然換到了銀寶珠」,要靠修改改回彩虹
  水滴(對照 bug #2 / docs/22)。
- **龍王城當機**:勇者父親 & 八頭大蛇處「當掉啦」,要修改才能過(對照 bug #3 / 五頭龍)。
- **巴拉摩斯打不死**:「大概是 BUG 的關係,很多人都過不了這裡」,建議用毒針或修改(對照 bug #1)。
- 精訊版相對 FC 版漏掉不少物品:大地盔甲(勇氣的洞窟)、某墓前種子等。

中盤:阿薩拉姆鎮 → 西方沙漠金字塔(魔法鑰匙、黃金爪;地下室禁魔法)+ 依西斯城(流星手環、祈禱
戒指)→ 羅馬利亞祠堂用魔法鑰匙 → 魯波加爾城(找胡椒,寫信給霍比特通行)→ 巴哈拉達(救胡椒店
女兒 & 男友,擊敗甘達特,換得胡椒)→ 魯波加爾國王給**船**。**達瑪**可記錄進度 & 轉職(轉賢者需先
到加爾那塔取領悟之書)。日邦格洞窟打**八頭大蛇(作者稱五頭龍)**取草薙之劍,日邦格本體取紫寶珠。

後盤(取船後):提頓村(綠寶珠需先與獄中人對話——精訊版差異、黑暗之燈)、耶貝亞(隱身草進入、
推三石取乾渴壺)、姆歐魯北海域用乾渴壺浮出祠堂取**最後的鑰匙**、朗錫爾神殿「勇氣的洞窟」(藍寶珠)、
海盜村(紅寶珠)、亞布塔(回音之笛)、斯吾村(雷杖)、新市鎮(交商人同伴開發)。冰島兩祠堂的
旅人之門到南美洲**沙曼歐沙城**(被冒充的國王、拉之鏡 & 貓衣、變身杖)。

終盤:六寶珠齊 → 朗錫爾西南冰島祠堂祭台 → 復活不死鳥拉米亞 → 龍女王城取**光珠** → 依西斯南
**巴拉摩斯城**。打贏巴拉摩斯後回阿里阿罕,**大魔王索馬**現身,地震震垮蓋亞那洞穴的牆 → 跳下進
**亞烈弗加特**(作者稱「一代大陸」/ 黑暗世界)。黑暗世界:拉達托姆城(太陽石)、洛特的洞窟(勇者
之盾)、達姆度拉(歐里哈爾昆金屬)、瑪依拉村(妖精之笛、賣金屬買王者之劍)、光之盔甲塔(魯比斯
守護)、美魯吉特南祠堂(雲雨杖)→ 利姆達爾南祠堂合成彩虹水滴 → 利姆達爾西北用彩虹水滴造橋 →
**龍王城** → 賢者之石、八頭大蛇 / 巴拉摩斯僵屍 / 龍僵屍 → **索馬**(先用光珠除黑暗之衣,夫巴哈 +
馬荷坎達 + 賢者之石)。全破後王者之劍→洛特之劍、光之盔甲→洛特之甲、勇者之盾→洛特之盾。

作者署名:**94/07 Dollar Cheng【孔方兄】**。

## 四、青衫詩客攻略(小邱,1995.09)

討論串後段收錄了 **青衫詩客 -- 小邱**(Chi'u I-Nan,1995.09.22)的條列式攻略,分光明世界 6 步、
黑暗世界數步。這位作者正是本專案 `./reference` 收錄的青衫攻略來源。其黑暗世界段落把當機點與存檔
修改寫得最清楚:

- 龍王城地下室往下遇勇者父親 & 五頭龍會當機,**遇見前存檔**,改 `DRAGON?.DAT`:
  `Displacement 000A → 0A`、`Displacement 0023 → 0A`;取檔後「不要往回走,繼續往前」。
- 下一層又一隻當機五頭龍,改 `Displacement 000C → 0A`、`Displacement 0025 → 0A`。
- 魔王(索馬)很難打:「等級 50 左右約要打上近百回合」,一開始用光之珠降魔王法力,每回合用賢者
  之石補血。

魔王打不死同樣要改 `DQ3.EXE`:`39 87 36 23 73 04 8B 87 36 23` → 第 3 與第 8 byte 改 `34`。

## 五、精訊版 BUG 與當年的修改法(一手對照表)

這是整份史料對反組譯專案**最有價值**的部分。當年玩家用 PCTOOLS / Norton DiskEdit 摸出來的
binary patch,和我們今天從 `DQ3.EXE` 反組譯出的 bug 是同一批。逐條對照如下:

| 當年修改法 | 檔案 | Find → Edit | 對應本專案 RE |
|---|---|---|---|
| **魔王打不死修正法** | DQ3.EXE | `39 87 36 23 73 04 8B 87 36 23` → 第3/第8 byte 改 `34` | **bug #1 巴拉摩斯打不死**(docs/18 §1,sub_aa69 file 0xbe02;`73`=jae 比較改 `34`=xor 系列) |
| **彩虹水滴製作法** | DRAGON?.DAT | 物品欄第一 byte=物品碼,改未使用欄(FFh)為 `75h` | **bug #2 彩虹水滴**(docs/22;彩虹水滴 id=0x75,合成誤產銀寶珠 0x6b) |
| **五頭龍大王避戰防當法(第一隻,歐魯狄加戰)** | DRAGON?.DAT | `Disp 000A→0A`、`Disp 0023→0A` | **bug #3 九頭龍 / 歐里狄加當機**(空 sprite 128/129;存檔改遭遇 id 規避崩潰) |
| **五頭龍大王避戰防當法(第二隻,索瑪前)** | DRAGON?.DAT | `Disp 000C→0A`、`Disp 0025→0A` | 同上,第二個觸發點 |
| **聲霸卡防當法** | DQ3.EXE | `BE E8 80` → `-- F0 77` | 音效初始化崩潰(Sound Blaster 偵測);非劇情 bug |
| **音效當機(可保留音樂)** | DQ3.EXE | `3D 00 00 74 1D 9A` → `90 90`;`3D 00 00 74 02 EB EA` → `EB` | 音效呼叫崩潰(`cmp ax,0` / far call) |
| **敵人消失法 / 不遇敵** | DQ3.EXE | `FE 0E F4 52 75 30 F7` → `90 90 90 90 EB` | 遇敵步數計數器(`dec byte [step]`)空轉;對照攻略「遇敵機率過高」 |

幾個觀察:

- **bug #1 / #2 / #3 三者** 在當年就被獨立發現並各有修改法,和我們反組譯結論(docs/18–22)
  完全一致。這是 RE 正確性的一個外部交叉驗證。
- **五頭龍大王(歐魯狄加 / 索瑪前)兩個觸發點** 對應 sprite 表 id 128(歐里狄加)/ 129(五頭龍
  大王),正是 remake 階段④ bug #3 待填的兩個空 sprite 槽。當年玩家是「改存檔避戰」繞過,我們的
  修法是 blit guard + 復原 sprite。
- **遇敵率過高** 不在我們列管的 7 個 bug 內,但攻略反覆抱怨,值得日後在 remake 對遭遇生成
  (sub_a7d5)校準時參考。

> ⚠ 著作權:上述 byte pattern 為當年公開流傳的修改資訊,作為 RE 對照引用;本專案不散佈遊戲本體。

## 六、存檔(DRAGON?.DAT)記憶體佈局

Paul Shih(1994.07.04,GAME-RPG)整理的「勇者鬥惡龍Ⅲ之存檔修改法」給出**每名角色的數值欄位
偏移**。這對 remake 的存檔 / 讀檔與隊員結構 RE 是一手資料。第一名角色 base = `0x0363`,每名角色
區塊 stride ≈ `0xFF`(255 byte):

| 欄位 | 第一人偏移 | 寬度 |
|---|---|---|
| 等級 | 0x0363 | 1 byte |
| HP | 0x0364–0365 | 2 byte |
| MP | 0x0366–0367 | 2 byte |
| 力量 | 0x0368–0369 | 2 byte |
| 攻擊 | 0x0370–0371 | 2 byte(派生值) |
| 耐力 | 0x0372–0373 | 2 byte |
| 守備 | 0x0374–0375 | 2 byte(派生值) |
| 速度 | 0x0378–0379 | 2 byte |
| 聰明 | 0x0380–0381 | 2 byte |
| 運氣 | 0x0382–0383 | 2 byte |
| 最大 HP | 0x0384–0385 | 2 byte |
| 最大 MP | 0x0386–0387 | 2 byte |
| 法術(魔法系) | 0x0388–0389 | 2 byte |
| 法術(僧侶系) | 0x0390–0391 | 2 byte |
| 道具欄 | 0x0400–0415 | 8 格 × 2 byte(`碼, 屬性`;屬性 80h=已裝備) |

第二人 base `0x0462`、第三人 `0x1:0049`、第四人 `0x1:0148`。金錢在 `0x0014–0017`。

對照本專案:

- **屬性順序** = 力量 / 耐力 / 速度 / 聰明 / 運氣 + HP / MP,正好對上升級訊息(D3TXT00 rec
  191–197:力量 / 耐力 / 速度 / 最大 HP / 最大 MP / 聰明度 / 運氣)與成長表 7 欄(docs/23)。
  這把 remake `dq3_member` 目前只存 6 屬性、缺「運氣」的缺口補實了——應補第 7 屬性。
- **攻擊 / 守備是派生值**(力量 + 武器、體力 + 防具),與基礎屬性分開存,呼應戰鬥傷害公式
  (docs/13)讀 atk/def 的設計。
- **道具欄每格 2 byte `{物品碼, 屬性}`**、屬性 bit7(80h)=已裝備,與我們 `dq3_inventory` /
  鑰匙門(docs/35 §八)讀道具碼的格式一致。
