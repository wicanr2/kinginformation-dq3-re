# 文字腳本格式:D3TXT*.TXT(遊戲對話 / 劇情文字)

精訊版 DQ3 的對話與劇情文字已完整解出。`assets_raw/` 內實際只有 **`D3TXT00.TXT` ~ `D3TXT09.TXT` 共 10 檔**(`01-asset-inventory.md` 舊記的「00~11、12 檔」為偵察階段誤估,以本檔為準)。所有 TXT 共用同一種容器格式,文字直接以 **2-byte glyph index** 引用 `D3TXT00.FON` 字模(見 `02-font-format.md`),不經 Big5、不經 `CHINA.FON`。

驗證方式:把每筆記錄的 glyph index 序列逐碼取 `D3TXT00.FON` 字模 render 成圖,即拼出**完全通順的繁體中文對話**(例:`槍棒`、`阿里阿罕是個被海所包圍的大陸`、`勇者洛特`)。驗證圖在 `assets_out/txtNN_full.png`,工具見下方。

## 容器結構(10 檔共用,已驗證)

```
+0x00   指標表(pointer table):N 個 16-bit LE offset
        ptr[0] 同時標示指標表結束(= 表大小,單位 byte)
ptr[0].. 記錄資料區:第 i 筆記錄 = data[ptr[i] .. ptr[i+1]]
        每筆記錄 = 一串 2-byte LE 值,以 0xffff 終止
        最後一個 ptr = filesize(結尾哨兵)
```

- 指標表項數 = `ptr[0] / 2`;有效記錄數 = 項數 − 1(最後一項為結尾哨兵)。
- 每筆記錄是一段 **2-byte little-endian 值序列**,通常以 `ff ff`(0xffff)結尾。
- 值有兩類:
  - **glyph index `< 1476`**:直接對應 `D3TXT00.FON` 第 N 個 16×16 字模。
  - **控制碼 `>= 0xffed`**:換行 / 換頁 / 動態插值占位(見下)。

## D3TXT00 的特殊首筆記錄(rec 0)

`D3TXT00` 第 0 筆(off=1520,139 個值)不是對話,而是 glyph index `0x00..0x85` 的近似順序列表(含少量重複),等同一張**字元集 / 字模索引表**——它列出本檔文字所用的 ASCII / 注音 / 符號區段。rec 1 起才是真正的道具名與訊息。

## 控制碼語意(由上下文與 render 對照確認)

文字流中夾雜的高值碼為控制碼,非字模:

| 碼 | 出現次數 | 語意 |
|---|---|---|
| `0xfffe` | 1,484 | **換行**(視窗內換行,後面幾乎必接 `0x0c 0x0c` 兩格縮排) |
| `0xfffc` | 318 | **換頁 / 訊息段落分隔**(下一段常以 `0x3b 0x3c` = `*「` 起頭) |
| `0xfffd` | 3 | 換行(變體) |
| `0xfffb` / `0xfffa` / `0xfff9` / `0xfff6` / `0xfff5` / `0xffed` | 共 ~280 | **動態插值占位**(`{VAR}`):引擎執行時填入主角名 / 道具名 / 金額等變數;render 時呈現為空白缺口 |

低索引字模中,標點與框線同樣是 `D3TXT00.FON` 的字模,常用對照(由 labeled atlas 確認):

| glyph index | 字 | | glyph index | 字 |
|---|---|---|---|---|
| `0x0c` | (空白 / 空格) | | `0x3a` | `?` |
| `0x37` | `,` | | `0x3b` | `*`(對話列表記號) |
| `0x38` | `。` | | `0x3c` | `「`(對話起始引號) |
| `0x39` | `!` | | `0x3d` | `:` |

`0x00..0x09` = `0-9`,`0x10..0x29` = `A-Z`,`0x41..0x6a` = 注音符號,`0x6b..0x72` = 職業字(勇戰武僧魔賢商遊),`0x73..0x76` = 羅馬數字 I~IV,`0x77` = ★,`0x86` 起 = 道具名漢字(檜木棒毒針銅劍…)。完整 atlas 見 `02-font-format.md` 與驗證圖 `assets_out/atlas_00-8f.png`。

## 與 Unicode 的對照(現況:未建)

精訊字型是**自訂內部碼**,glyph index 與 Unicode / Big5 無現成對照表,`.CF1` / `.CF2` 也只是字型子集標籤(見下),不含 index→Unicode 映射。因此本階段:

- **可正確 render 原文**(決定性 pass:render 出的圖即原版畫面文字),此為主要成果。
- **尚未**建立 glyph index → Unicode 的全表(需逐字 OCR 1,476 個字模,屬後續工作)。
- 結構化、可追溯的中繼資料以 glyph index 形式保存,不假裝是 Unicode。

## D3TXT00.CF1 / .CF2 — 字型子集標籤(非 index→Unicode 碼表)

| 檔 | 內容(ASCII)|
|---|---|
| `D3TXT00.CF1` | `CR CS CW NA NU OT A4 A3 A2 A1 U4 U3 U2 U1 O4 O3 O2 O1` + CRLF + `1A`(EOF) |
| `D3TXT00.CF2` | `AN` + CRLF + `1A`(EOF) |

兩檔皆為 ASCII 純文字 + DOS CRLF + `0x1A` 結尾,是 **2 字母代號的清單**,研判為字型烘製 / 字集分區的**區段標籤**(如 `A4 A3 A2 A1` / `U4..U1` / `O4..O1` 像注音四聲或母音分組,`NA NU OT CR CS CW` 像字集類別代號),**不是** glyph index → 字碼的對照表。對解碼 TXT 非必要(TXT 直接用 glyph index),保留待字型製作流程進一步釐清。

## 工具

全部在 docker uv venv 跑(`tools/dockrun.sh`):

| 工具 | 用途 |
|---|---|
| `tools/text_render.py <TXT> <out.png> [start] [count]` | render 指定 TXT 的記錄區間成圖(肉眼驗證中文) |
| `tools/text_dump.py` | 一次解析全部 10 檔 → `assets_out/txtNN_full.png`(全檔驗證圖)+ `docs/data/d3txt_codes.json`(結構化 glyph index) |
| `tools/text_atlas.py <start> <count> <out.png>` | render 標註索引的字模 atlas(辨識低索引標點 / 控制字模) |
| `tools/text_export.py` | 把記錄轉成標註版文字 dump(控制碼語意化、glyph 以 `#索引` 表示)→ `docs/data/D3TXTNN_codes.txt` |

驗證圖(render 出的中文)輸出在 `assets_out/`(git 已排除,因含版權畫面內容)。
