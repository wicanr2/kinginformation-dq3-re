# 消耗品使用效果(#3)

> 道具欄「使用」實際生效。記錄消耗品 id 的 RE 依據、效果種類、以及哪些數值是 RE 確鑿、
> 哪些採經典 DQ3 值(誠實標註,不自由臆造)。

## 消耗品 id:商店貨架交叉驗證(RE 確鑿)

ITEM.DAT 是**裝備資料**(每筆 7 byte:b0 攻、b1 防、+2 價、+4 類別槽、b6 職業遮罩),
**不含**消耗品使用效果。但消耗品的 **id** 可由商店貨架(docs/40,`dq3_shop_itempool`)
與 ITEM.DAT 價格交叉鎖定——精訊版商店價**全中經典 DQ3 值**:

| id | 名稱 | ITEM.DAT 價 | 經典 DQ3 | 效果種類 |
|---|---|---|---|---|
| 0x41 | 藥草 | 8 | やくそう 8G | 回 HP |
| 0x42 | 驅毒草 | 10 | どくけしそう 10G | 解毒 |
| 0x43 | 蓋美拉翅膀 | 25 | キメラのつばさ 25G | 回最後城鎮/出洞 |
| 0x44 | 聖水 | 20 | せいすい 20G | 暫時驅弱敵 |
| 0x45 | 滿月草 | 30 | まんげつそう 30G | 解麻痺 |

佐證:商店貨架出現順序(docs/40 row1「藥草、驅毒草、聖水、蓋美拉翅膀」)= 池內
`0x41,0x42,0x44,0x43,0x45` 一致 → id↔名稱對應鎖定,非臆測。

## 效果數值的來源誠實標註

效果「**種類**」是 RE 事實(上表)。需要**數值**的量,落在未定位的 use-effect 路徑:
- 道具使用**不以 id 直接分支**(EXE 內 `cmp al,0x41` 兩處經反組譯確認為 `[0x41xx]` 位址
  byte 的假陽性,非真比較)。治療量應在 use-effect 派發(疑與 ホイミ 回復共路徑),
  而**咒文威力表本專案已確認未定位、不憑 BBS 湊**(docs/13:312)。
- 故藥草治療量、聖水步數採**經典 DQ3 值**並在此標註:
  - `DQ3_HERB_HEAL = 30`(經典藥草固定回復;商店價已證精訊=經典資料,取經典值合理)。
  - `DQ3_HOLY_STEPS = 64`(聖水驅敵步數,經典量級)。
  - 兩者若日後 RE 定位到精訊實際值,於 `dq3_item_use.h` 回填即可。

## 落地(`dq3_item_use.{h,c}` + main.c)

- `dq3_item_use_kind(id)`:id → 效果種類(上表;裝備/鑰匙/劇情物 → NONE)。
- `dq3_item_use_heal(member, id)`:cur_hp += 30,封頂於 max(stat[HP]);陣亡(cur_hp==0)
  不可用藥草(需教會復活);已滿回 0。用 #task1 的持久 HP。
- main.c `apply_item_use`:HEAL_HP 治第一個受傷隊員 + 消耗;RETURN_TOWN/REPEL 消耗後交世界層;
  CURE_*(解毒/解麻痺)因**狀態系統未建**(docs/47 #5)→ 報「無對應異常」且**不消耗**(對齊原版)。
- 世界層:蓋美拉翅膀 → 回地表;聖水 → `repel` 步數,期間 overworld 不觸發隨機遇敵(每步遞減)。
