# 注音姓名輸入子系統(RE → C)— 精訊中文化技術亮點

精訊版 DQ3 在新遊戲建立主角時,沒有沿用日版的假名表,也不是西式字母輸入,而是內建了一套
**注音(Zhuyin / ㄅㄆㄇㄈ)輸入法** —— 玩家用方向鍵在注音鍵盤上選聲母、韻母、聲調拼出一個
中文字,引擎查出對應國字(同音字多時還會跳「選取國字」候選窗),寫進角色姓名。這是這份未發售
中文版裡最值得記錄的本地化技術:一套完整的螢幕注音輸入子系統,做在 1993 年的 DOS real-mode
組語遊戲引擎裡。

本文件把這個畫面的 handler 從 `DQ3.EXE` 反組譯出來:鍵盤版面、游標移動、組字流程、功能列、
名字緩衝、以及最關鍵的「完成」按鍵序列(供 DOSBox 自動化推過開場)。實機畫面見
`work/dosbox/live_03_name_entry.png`(版權畫面,不入版控)。

對應反編譯:`re/nameinput.c` + `re/nameinput.h`。工具:`tools/re_name_dis.py`(docker capstone)。

```
docker run --rm -v "$PWD":/work -w /work ghcr.io/astral-sh/uv:python3.12-bookworm-slim \
  bash -c "uv venv -q /tmp/venv && . /tmp/venv/bin/activate && uv pip install -q capstone \
           && python tools/re_name_dis.py"          # 全部 handler + 對照表
           # python tools/re_name_dis.py layout      # 只印注音鍵盤 cell→碼表
```

## 怎麼找到這個 handler

開機流程(`seed_and_names` / sub_6f4b)只負責「BIOS tick 亂數種子 + 預設角色名表」,不含
姓名輸入畫面 —— 姓名輸入是「遊戲開始」後角色建立時才進的獨立 modal。定位方法:

1. 畫面上「輸入姓名 / 輸入注音 / 英數 / 前進 / 後退 / 取消 / 完成」都是中文,在 `D3TXT00.TXT`
   裡找這些字串的記錄號:命中 **rec 451–456**(注音 / 英數鍵盤版面 + 組字框 + 選取國字)。
2. 在 `DQ3.EXE` 找載入這些記錄號的指令(`mov di, 451..456`):全部落在 **file 0x2202–0x23c0
   = seg0 0x0e92–0x1050** 這一小段 → 姓名輸入 handler 就在這裡。
3. 函式圖(`docs/data/exe_funcs.json`)切出這段的函式邊界與 caller 鏈:
   角色建立 modal **sub_0854** → dispatcher **sub_0d17** → {英數 sub_0e8e / 注音 sub_0f5b}
   → 共用 grid 導航 **sub_23f7**。

## 子系統結構

```
sub_0854   角色建立 modal(設 [0x5082] 角色結構、[0x5077]=1 隊伍人數)
  └─ sub_0d17  ni_dispatch     依模式旗標 [0x26fc] 分派三畫面,迴圈到「完成 / 取消」
       ├─ sub_0dc8  ni_fn_list     功能列(英數/前進/後退/取消/完成)
       ├─ sub_0e8e  ni_alnum       英數鍵盤(0-9 A-Z 羅馬數字 ★ OK)
       └─ sub_0f5b  ni_zhuyin      注音鍵盤(ㄅㄆㄇㄈ… 聲調 OK)
            ├─ sub_23f7  ni_grid_nav      方向鍵 + Space/Enter 選格(三畫面共用)
            ├─ sub_2623  ni_compose_put   注音符號 → 組字緩衝
            └─ sub_2654  ni_compose_resolve 組字 → 注音引擎查國字(選取國字候選窗)
```

| seg0 | file | 名稱 | 角色 |
|---|---|---|---|
| 0x0d17 | 0x2087 | ni_dispatch | 主控:三畫面分派 + 名字定案 |
| 0x0da4 | 0x2114 | ni_count_name | 數名字非空字數 → [0x270a] |
| 0x0dc8 | 0x2138 | ni_fn_list | 功能列 5 列導航 |
| 0x0e55 | 0x21c5 | ni_redraw_cursor | 名字列游標 + 滑鼠命中區 |
| 0x0e8e | 0x21fe | ni_alnum | 英數鍵盤 |
| 0x0f5b | 0x22cb | ni_zhuyin | 注音鍵盤 + 組字 |
| 0x23f7 | 0x23f7 | ni_grid_nav | grid 鍵盤導航迴圈 |
| 0x2623 | 0x2623 | ni_compose_put | 放注音進組字緩衝 |
| 0x2654 | 0x2654 | ni_compose_resolve | 查國字 + 選取國字候選窗 |

## 注音鍵盤版面(rec 452)

版面資料直接是 `D3TXT00` 的記錄,以文字繪製器 `lcall 111b:0214`(DI=記錄號)整段畫出。
注音鍵盤 = **5 列 × 9 欄**,欄優先排列;每格的 glyph index 直接取 `D3TXT00.FON` 字模
(注音符號字模在 glyph index **65–101**,聲調 **102–105**,見 docs/02 / docs/03)。

實機版面(rec 452 解碼,左為 9 欄注音格,右為功能列):

```
ㄅ ㄊ ㄏ ㄔ ㄙ ㄞ ㄣ ㄨ ˇ │ 英數
ㄆ ㄋ ㄐ ㄕ ㄚ ㄟ ㄤ ㄩ ˋ │ 前進
ㄇ ㄌ ㄑ ㄖ ㄛ ㄠ ㄥ ˙ ← │ 後退
ㄈ ㄍ ㄒ ㄗ ㄜ ㄡ ㄦ □ ⊙ │ 取消
ㄉ ㄎ ㄓ ㄘ ㄝ ㄢ ㄧ ˊ OK│ 完成
```

> 注意:資料是「逐列」存的,但 grid 游標換算成 cell 是「**欄優先**」(`cell = col*5 + row`)。
> 也就是第 1 欄 = ㄅㄆㄇㄈㄉ(聲母 1–5)、第 2 欄 = ㄊㄋㄌㄍㄎ(聲母 6–10)…注音符號按
> 「聲母 → 韻母 → 介音 → 聲調」順序填 45 格,右下角是 OK。英數模式(rec 453)同框換成
> 0-9 / A-Z / 羅馬數字 / ★☆∞⊙ / OK,功能列首列改顯示「注音」(切回注音)。

### cell → 注音碼對照表(DS:0x2758)

每格 1 byte,編碼 `(group<<5) | code`。`ni_compose_put` 以 `group` 當索引,把 `code` 寫進
組字緩衝 `[0x2730 + group]`:

| group | 注音類 | 組字緩衝位址 | 格數 | code 範圍 |
|---|---|---|---|---|
| 0 | 聲母(ㄅㄆㄇㄈ…) | [0x2730] | 21(cell 0–20) | 1–21 |
| 2 | 韻母(ㄚㄛㄜ…) | [0x2732] | 13(cell 21–33) | 1–13 |
| 1 | 介音(ㄧㄨㄩ) | [0x2731] | 3(cell 34–36) | 1–3 |
| 3 | 聲調(˙ˊˇˋ+一聲) | [0x2733] | 5(cell 37–41) | 0–4 |

cell 42 = OK、43 = 切英數、44 = 離開(這三格不走組字,由 `ni_zhuyin` 的 cell 值分支處理)。

## 游標移動與選格(ni_grid_nav, sub_23f7)

注音 / 英數 / 選取國字三畫面共用同一個導航迴圈。每輪 `lcall 1104:8a` 讀鍵盤 scancode(AH):

| scancode | 鍵 | 動作 |
|---|---|---|
| 0x48 | ↑ | `[0x26fe] -= 欄寬(9)`;< 0 環繞到底列(`+= 45`) |
| 0x50 | ↓ | `[0x26fe] += 9`;≥ 45 環繞回頂列(`-= 45`) |
| 0x4b | ← | `[0x26fe]--`;< 0 環繞(`+= 45`) |
| 0x4d | → | `[0x26fe]++`;≥ 45 環繞(`-= 45`) |
| **0x39 / 0x1c** | **Space / Enter** | **★ 選取目前游標格 ★**(回傳 g_action_result=0) |
| 0x64 | (滑鼠移動) | 取消本次(回傳 1) |
| 0x65 | (滑鼠點擊) | 以滑鼠座標換算命中格 → 視為選取 |

游標是 raw index `[0x26fe]`(0–44);選格後換算成 cell:`raw/9 = row, raw%9 = col,
cell = col*5 + row`。**Space 與 Enter 等價,都是「選取游標所在格」**,並非「確定整個名字」。

## 組字流程(注音 → 國字)

`ni_zhuyin` 拿到選取的 cell 後分四路:

1. **cell < 37(注音 / 聲調符號格)**:`ni_compose_put` 把該注音碼依 group 填進組字緩衝
   `[0x2730]`聲母 / `[0x2731]`介音 / `[0x2732]`韻母 / `[0x2733]`聲調,並在「輸入注音」框
   即時畫出已選注音;回 grid 繼續選下一個注音。
2. **37 ≤ cell < 42(組字完成格)**:`ni_compose_resolve` 把四個注音交給**注音引擎 seg 0x11c4**:
   - `lcall 11c4:0x27`:in `AH`聲母 `AL`介音 `BH`韻母 `BL`聲調 → out `DX:AX` 國字碼、
     `BL` = 同音候選字數。
   - 候選數 = 0 → 查無此音(`beep` si=0x1e1),回頭重組。
   - 候選數 > 0 → 開**「選取國字」候選窗(rec 454)**:`lcall 11c4:0x81` 逐個取同音字畫進窗,
     再用同一個 `ni_grid_nav` 讓玩家用方向鍵 + Space/Enter 挑一個;選定索引交 `lcall 11c4:0x97`
     算出最終國字碼。
   - 選定國字寫進名字緩衝,`[0x270a]`(字數)+1,**上限 4 字**(滿 4 字停在最後一格)。
3. **cell == 42(OK)**:重畫「輸入注音」框、清組字緩衝,繼續輸入下一字。
4. **cell == 43(切英數)/ 44(離開)**:設模式旗標切換 / 離開。

名字緩衝 **DS:0x2710**:每字佔 2 word(`[0x2710]`碼高位 / `[0x2712]`碼低位),最多 4 字。
字數在 `[0x270a]`。dispatcher 結束後由角色建立 modal `sub_0854` 把這個緩衝搬進角色結構
(`[0x5082]` 區),成為主角姓名 —— 對應 `seed_and_names`(sub_6f4b)平時填的預設名表,
此處改由玩家手動拼出。

## 功能列(ni_fn_list, sub_0dc8)

右欄 5 列,`win_nav(5)` 回 `[0x722]`(1-based)選中列:

| [0x722] | 功能鍵 | 動作 |
|---|---|---|
| 1 | 英數 / 注音 | 切換鍵盤模式(翻 [0x26fc] bit1) |
| 2 | 前進 | 名字游標往後一格(`[0x270a]++`,不超過已輸入長度) |
| 3 | 後退 | 名字游標往前一格(`[0x270a]--`,刪/退) |
| 4 | 取消 | 設 `[0x726]=1` → dispatcher 中止(整個姓名輸入作廢) |
| 5(else) | **完成** | **設 `[0x26fc] |= 8`(離開旗標)→ dispatcher 下一輪定案名字** |

`ni_dispatch` 的離開條件:`[0x26fc] & 8`(完成旗標)且 `[0x270a] ≥ 1`(名字非空)才放行;
**名字空時按完成不會離開**(`[0x26fc]` 被清掉、繼續輸入)。

## 「完成名字輸入」的確切按鍵序列(供 DOSBox 自動化)

要程式化推過這道必填關卡(headless 自動化卡關的根因,見 docs/14),最短可靠序列:

1. **輸入至少一個字**(名字不可為空,否則「完成」無效):
   - 注音模式:在注音格上用方向鍵移到一個聲母 → `Space`(或 `Enter`)選它 →(視需要)移到
     韻母 / 聲調再 `Space` → 移到「組字完成格」`Space`;若跳出「選取國字」候選窗,方向鍵選字
     再 `Space`。
   - **更快**:切到英數模式(下一步),英數模式選一個字元格(如數字 0)= 一個字,免組字。
2. **切到功能列**:在鍵盤格內把游標移到右下「離開 / 切功能列」格並 `Space`,或直接讓游標進入
   功能列區。功能列為右欄 5 列(英數/前進/後退/取消/**完成**)。
3. **選「完成」**:在功能列上方向鍵移到第 5 列「完成」→ `Space` / `Enter`。
   - 對應 `[0x722]=5`,設 `[0x26fc] |= 8`;dispatcher 確認名字非空後即離開,進城鎮 / 世界圖。

xdotool 對照(headless,scancode 由 DOSBox 轉成方向鍵 + space/return):

```
# 例:英數模式打一個字 + 完成
xdotool key Down Down ... space        # grid 內移到字元格,選 1 字
xdotool key Right                       # 移向功能列(或選「切功能列」格)
xdotool key Down Down Down Down space   # 功能列移到「完成」(第5列)按下
```

> 關鍵:**Space / Enter 永遠是「選取游標所在格」,不是「直接完成」**;真正的「完成」是功能列
> 第 5 列那一格。先前自動化只在注音格內打轉、按 Enter 沒反應,就是因為沒移到功能列的完成格。
> 另一條更省事的路:用主選單「載入進度」走既有存檔略過開場(docs/14 (b) 方案)。

## 殘留 / 待釐清

- 注音引擎 **seg 0x11c4** 的內部查表結構(`11c4:0x27` 注音碼 → 國字碼的對照、`0x81` 候選字
  列舉、`0x97` 定案算式)尚未反組譯;目前以「黑箱輸入輸出契約」表達。解開後可建「注音 →
  內部國字碼 → glyph index」完整對照,與 docs/02 的 CHINA.FON 母字庫字碼對接。
- `sub_0854` 把名字緩衝 `[0x2710]` 搬進角色結構 `[0x5082]` 的精確欄位偏移(姓名在角色結構中
  的位置)待逐欄反組譯角色資料格式時補上。
- 滑鼠路徑(AH=0x64/0x65 + int33 命中區)只標出語意骨架;精訊讓姓名輸入同時支援鍵盤與滑鼠,
  細節待補。
- cell 42/43/44(OK / 切英數 / 離開)在英數模式的對應(rec 453 右下角)與注音模式一致性已交叉
  驗證;聲調 group 的 code 0(一聲/輕聲)語意待對 CHINA.FON 聲調字模確認。
